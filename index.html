The 4-step ranking pipeline

Every recommendation flows through the same deterministic assembly line: capture the user's profile, score it against the catalog, rank by relevance, and truncate before choice overload sets in.

01 / Ingestion

Capture the profile

Raw skill labels are normalised to a shared vocabulary so "Frontend Dev" and "Web Design" map to the same vector dimension.

02 / Scoring

TF-IDF weighting

Common terms like "Git" are dampened. Rare specialist skills like "Solidity" carry full weight. Then similarity is computed as the angle between vectors.

03 / Sorting

Rank by relevance

All 20 career paths receive a similarity score between 0 and 1. The catalog is re-ordered highest-first in a single O(n log n) pass.

04 / Filtering

Top-3 output

The list is truncated to the three strongest matches — enough signal to act on without triggering decision fatigue.

Under the Hood

The math behind the match

Built from scratch without ML libraries. Every algorithm is implemented in pure JavaScript, running entirely client-side.

Term Frequency

TF weighting

Each term's raw count is divided by document length, normalising short vs. long skill sets.

// term frequency
tf(t, d) = count(t, d) / |d|
Inverse Document Frequency

IDF dampening

Universal skills like "Git" appear in most roles — IDF penalises common terms and boosts rare, specialist ones.

// inverse doc frequency
idf(t) = log((N+1) / (df+1)) + 1
Cosine Similarity

Vector angle

The dot product of two TF-IDF vectors normalised by their magnitudes. A perfect match returns cos θ = 1.

// cosine similarity
sim(A, B) = (A·B) / (‖A‖ · ‖B‖)
